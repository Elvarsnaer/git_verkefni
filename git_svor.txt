Git spurningar og svör
1. Hvað gera eftirfarandi Linux skipanir?
	cd = breytir um directory
	cd ~ = breytir yfir í home directory
	ls = birtir lista um efni í file-um og directories
	pwd = birtir fulla nafnið á working-directory
	mkdir = býr til directories og files

2. Hvað gera eftirfarandi git skipanir og hvers vegna eru þær gagnlegar?
	git clone = klónar repository í nýtt directory
	git log = sýnir commit log
	git status = sýnir working tree status
	git diff = sýnir breytingar á milli commits, commit og working tree, o.s.f.v.
	git checkout = skiptir um branch eða restorar working tree files

3. Hver er munurinn á eftirfarandi git skipunum:
	a) git diff = sýnir breytingar á milli commits, commit og working tree, o.s.f.v.
	b) git diff --staged = sýnir hvað hefur verið stage-að fyrir næsta commit
	c) git diff commit1 commit2 = sýnir muninn á tveimur commits

4. 	Hvað er útgáfustýring (e.version control)? = a system that records changes to a file or set of files over time so that you can recall specific versions later.

5.	Hverjir eru helstu kostir við að nota GIT? = margir geta unnið á sama kóða og version control.

6.	Hversu oft telur þú að eigi að gera commit í verkefni, rökstuddu? = þegar maður er búinn að klára einhvern sérstakan bita af kóðanum, það er betra að gera fleiri en fáa commit svo þú þarft að vinna þig minna upp ef þú klúðrar einhverju.

7.	Hvað er "Working directory"?" "Staging area"?" og "Repository" í GIT? = Working directory is the directory with your source files under git control, 
                                                                                The staging area is a file that stores information about what will go into your next commit., 
                                                                                The purpose of Git is to manage a project, or a set of files, as they change over time. Git stores this information in a data structure called a repository.

8.	Hvenær er sniðugt að nota greinar (branches)? = When you want to add a new feature or fix a bug—no matter how big or how small—you spawn a new branch to encapsulate your changes.
